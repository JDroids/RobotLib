addParallel

fun addParallel(command: Command): Unit

Adds a new child Command to the group. The Command will be started after all the previously added commands.

Instead of waiting for the child to finish, a CommandGroup will have it run at the same time as the subsequent commands. The child will run until either it finishes, a new child with conflicting commands is started, or the main sequence runs a Command with conflicting requirements. In the latter two cases, the child will be canceled even if it says it can't be interrupted.

Note that any requirement the given Command has will be added to the group. For this reason, a command's requirements can not be changed after being added to a group.

It is recommended that this method be called in the constructor.

Parameters

command - the command to be added

Exceptions

IllegalUseOfCommandException - if the group has been started before or been given to another group
